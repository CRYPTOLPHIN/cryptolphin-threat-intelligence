# Cryptolphin Security Architecture

## Overview

Cryptolphin follows a privacy-first security model designed to detect threats while minimising unnecessary data collection.

---

## Core Security Layers

### Threat Intelligence Layer

Maintains verified intelligence covering:

- Scam wallet addresses
- Phishing domains
- Malicious infrastructure
- Social engineering indicators

---

### Local Detection Layer

Where possible, detection occurs locally within the user's browser.

Benefits:

- Reduced data exposure
- Faster response
- Improved privacy

---

### Verification Layer

Threat information is reviewed using multiple indicators.

Sources may include:

- Security research
- Blockchain analysis
- Public reports
- Community submissions

---

## Design Principles

### Prevention Over Recovery

Blockchain transactions are irreversible.

The strongest security measure is preventing interaction with malicious infrastructure before damage occurs.

### Privacy By Design

Cryptolphin avoids collecting unnecessary user information.

Security should not require surveillance.
